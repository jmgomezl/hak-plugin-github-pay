How it works

Self-sovereign identity, immutable policy, automatic settlement — in four steps.

01

Register

A contributor maps their GitHub handle → Hedera account on the public IDENTITIES topic. Any HAK plugin can reuse it.

02

Set policy + cap

A repo admin seals label → HBAR rules and spending ceilings on the POLICIES topic — a submitKey-locked, immutable audit trail.

03

Merge the PR

The merge is the approval. The agent validates an HMAC webhook, enforces idempotency & caps, and transfers HBAR.

04

Receipt sealed

A signed receipt lands on the RECEIPTS topic — verifiable forever on Hashscan.

Enterprise controls — enforced, not promised

Every payout runs through real, verifiable financial controls.

🔐

HMAC webhook validation

Every GitHub payload is verified against X-Hub-Signature-256 (constant-time) before anything is processed.

♻️

Idempotency

Dedup on the PR number across a durable HCS topic + a local guard. Webhook retries never double-pay.

📊

Spending caps

Monthly + per-contributor ceilings, read from the POLICIES topic and enforced before every transfer.

🗝️

Admin / payer key separation

The POLICIES topic is submitKey-locked. The key that pays cannot raise its own cap — proven on-chain.
